# Beauty Academy Website Constitution

## Core Principles

### I. User-Centric Design
The website must prioritize the needs of the target audience (youth aged 18-35 seeking beauty education). All design decisions, features, and content must serve the primary goals of generating leads and student applications. The interface must be intuitive, accessible, and reflect the professional, modern, and energetic brand identity of "Crafting careers in beauty".

### II. Lead Generation Focus
Every page and feature must contribute to the business objectives of achieving 10 student applications per week and 50 enquiries per week. This includes prominent placement of application forms, clear calls-to-action, and streamlined user journeys from awareness to conversion.

### III. Mobile-First Responsive Design
Given the target demographic and potential users in remote areas, the website must be fully responsive and optimized for mobile devices. All functionality must work seamlessly across devices to ensure accessibility for all potential students.

### IV. Accessibility & Performance
The website must load quickly and be accessible to users with disabilities. This includes proper semantic HTML, alt text for images, keyboard navigation support, and fast loading times to prevent bounce rates and improve SEO.

### V. Content Accuracy & Brand Alignment
All content must accurately represent the beauty academy's services, accreditation partnerships, and unique value propositions. Visual design and messaging must consistently reflect the brand identity and maintain the professional, trustworthy, and elegant tone specified by the client.

## Technical Standards

The website will be built using the Nuxt.js framework with Vue 3, leveraging the proven architecture from the Her Luxe Vault blueprint. This includes using TailwindCSS for styling, implementing dark/light mode capabilities, and following component-based architecture patterns. The technology stack must support SEO optimization, server-side rendering, and static generation capabilities.

## Quality Assurance

All features must undergo thorough testing including unit tests for components, integration tests for forms and user flows, and end-to-end tests for critical conversion paths. The website must be tested across multiple browsers and devices before deployment. Performance metrics must meet industry standards for Core Web Vitals and accessibility scores.

## Governance

This constitution governs all development decisions for the beauty academy website project. All PRs and reviews must verify compliance with these principles. Any proposed changes to core functionality or design direction must reference this constitution and justify alignment with the stated principles. Amendments to this constitution require documentation and approval from project stakeholders.
